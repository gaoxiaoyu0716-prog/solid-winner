01 项目概览

从普通商品售卖，到抽取式 ACG 消费体验

微博具备天然的 ACG 内容和超话圈层优势，但衍生品售卖在早期更偏普通商品交易，缺少抽赏、抽卡等能激发复购和惊喜感的玩法，商品售卖效果不如预期。

因此，项目希望通过「欧力赏」和「抽卡机」两个产品方向，补齐微博 ACG 衍生品从浏览、购买、开奖、资产沉淀到分享传播的完整链路，把 IP 内容兴趣转化为更稳定的消费行为。

设计目标

通过品牌化抽赏平台、抽卡体验升级和跨产品交互规范，提升用户购买信任、玩法理解效率、支付转化与复购动机。

从 0 到 1

欧力赏

依托微博生态的品牌化抽赏平台，围绕赏池、赏箱、赏品、订单、开奖和背包搭建完整抽赏交易链路。

已有产品升级

抽卡机

完成从 H5 到小程序的迁移，重构卡包详情页、抽卡反馈、卡牌收藏与分享链路。

设计沉淀

ACG 交互规范

基于微博主端规范，统一欧力赏与抽卡机相关页面的信息层级、交互模式和视觉风格。

02 关键问题

抽取式消费的难点不在“随机”，而在信任、理解和复购

问题 1

信任成本高：用户需要确认品牌、IP 与正版背书

ACG 衍生品消费高度依赖 IP 和授权信任。如果页面只突出抽奖刺激，而没有品牌、IP、概率和库存等信息，用户容易担心盗版、履约和公平性。

问题 2

玩法层级复杂：赏池、赏箱、赏别、赏品需要被清晰解释

抽赏不是普通商品详情页。用户需要理解买的是哪个赏箱、可能抽到哪些赏品、概率如何变化、Last 赏如何触发，以及抽中后商品如何流转。

问题 3

转化链路长：从浏览到支付中间存在多次决策损耗

用户需要先看 IP、看奖品、理解规则、判断价格和优惠，再进入下单支付。任何一个节点信息不清晰，都会造成下单或支付环节流失。

问题 4

多产品体验割裂：欧力赏与抽卡机需要统一消费语境

欧力赏和抽卡机玩法不同，但都属于微博 ACG 购买场景。若视觉风格、信息层级和抽取反馈不一致，会增加用户在不同产品间切换的适应成本。

设计判断

这个项目要解决的不是“做一个抽奖页面”，而是把 IP 信任、玩法理解、支付转化、资产沉淀和分享复购组织成一个完整交易闭环。

03 机会判断

微博的优势不是单纯售卖，而是 IP 圈层与社交传播

我在项目早期对一番赏 Online、B 站魔力赏、LOFTER 元气赏等产品进行基础分析。不同平台的抽赏能力各有侧重：一番赏 Online 强调正版授权和沉浸体验，B 站魔力赏依托站内生态与粉丝经济，LOFTER 元气赏更偏小众兴趣圈层和轻量化运营。

一番赏 Online

正版授权 / 沉浸抽赏

启发：购买前需要建立品牌、IP 与正版授权信任。

B站魔力赏

内容生态 / 粉丝消费

启发：需要把站内内容关系和购买链路自然衔接。

LOFTER 元气赏

轻量运营 / 小众圈层

启发：轻量化启动适合验证初期业务模型。

微博更适合做「品牌合作 + IP 圈层 + 抽取式购买」的 ACG 衍生品消费场景。

用品牌和 IP 双背书降低用户购买顾虑

用超话和内容关系承接 ACG 圈层兴趣

用抽赏和抽卡玩法提升复购刺激

用背包、图鉴和分享延长消费链路

04 设计策略

用“品牌信任 + 玩法转化 + 资产沉淀”搭建 ACG 消费闭环

抽赏和抽卡表面上是随机玩法，但真正影响用户决策的是：用户是否相信商品价值、是否理解玩法规则、是否感受到抽中结果的惊喜，以及购买后是否有继续互动和复购的理由。

因此，我把设计策略拆成三个方向：先建立品牌与 IP 信任，再让用户以低成本理解玩法并完成支付，最后把抽中结果沉淀为可管理、可展示、可分享的用户资产。

策略 01

品牌信任

在购买前展示品牌、IP、赏品、概率、库存和规则，让用户明确商品来源和抽取边界。

策略 02

玩法转化

把赏池详情页和卡包详情页作为转化中枢，前置核心操作、优惠信息和低门槛试玩。

策略 03

资产沉淀

通过背包、图鉴、翻卡和分享，让一次购买结果继续产生收藏、展示和传播价值。

05 核心方案

从新产品搭建到跨产品体验沉淀

欧力赏一期

欧力赏一期：搭建从 0 到 1 的抽赏交易链路

让用户从浏览赏池到支付开奖都能形成完整闭环

一期重点是先跑通完整抽赏流程。我负责赏池列表、赏池详情、赏品详情、背包、确认订单、支付购买和开奖结果等核心页面交互，确保复杂的抽赏规则能被用户逐步理解。

问题

抽赏产品层级复杂，如果直接堆叠规则，用户难以理解买什么、抽什么、抽中后去哪里看。

设计

按“看池子 — 看奖品 — 试玩法 — 下单 — 开奖 — 背包”的顺序搭建主链路。

价值

让抽赏从一个规则复杂的玩法，转化为可理解、可支付、可管理的交易产品。

核心转化页

赏池详情页：把复杂玩法组织成转化中枢

查看赏池、免费试玩、切换赏箱、查看赏品详情与购买决策都在这里发生

赏池详情页承担欧力赏最重要的转化功能。我将规则解释、奖品展示、概率库存、试玩入口和购买操作按用户决策顺序组织，避免用户在大量信息中迷失。

问题

用户需要同时理解 IP、赏箱、赏别、赏品、概率、价格和支付方式，信息密度高。

设计

将页面组织为“先建立信任，再理解奖品，再低成本试玩，最后购买”的连续路径。

价值

提升用户对玩法和商品的理解效率，为下单支付建立充分决策依据。

持续迭代

欧力赏迭代：补齐运营、优惠和履约能力

从“能买”继续迭代到“更好卖、更好管、更好发”

一期上线后，项目进入持续迭代。我继续跟进优惠券、折扣价、IP 筛选、商家卡片、STAGE 赏、分享、自动发货私信触达和后台配置等需求。由于 UI 资源紧张，我也参与了部分 UI 设计。

迭代重点

把一期验证过的购买链路继续扩展为可运营、可配置、可触达的长期商品体系。

抽卡机升级

抽卡机升级：从 H5 迁移到小程序，并重构核心转化页

保持品牌与体验一致，同时提升下单后的支付完成率

抽卡机是已有产品升级。我在迁移过程中重构卡包详情页，将抽卡相关操作、优惠券信息和购买入口前置；同时支持详情页左右滑切换卡包，降低跳转频次，并通过稀有度、图鉴、翻卡和分享强化收藏价值。

问题

原有详情页信息和操作分散，用户需要滑动或跳转才能完成抽卡决策。

设计

重构详情页布局，外露抽卡操作、优惠券信息和卡包切换能力。

价值

降低浏览到支付的路径损耗，同时增强抽卡结果的仪式感和分享动机。

规范沉淀

ACG 交互规范：从单点页面设计沉淀为跨产品标准

统一欧力赏 / 抽卡机相关页面的视觉风格、信息层级与交互模式

两个产品完成后，我基于微博主端规范，提炼 ACG 衍生品相关页面的交互规范，覆盖商品与 IP 信息展示、购买与抽取反馈、背包与图鉴资产、运营位和分享链路，降低后续重复设计成本。

规范的核心目标不是限制页面，而是让不同 ACG 消费产品都能在统一框架下扩展，帮助用户获得熟悉、连贯的购买体验。

规范 01

信息层级

统一 IP、品牌、商品、概率、库存、价格、优惠等高敏感信息的展示优先级。

规范 02

抽取反馈

统一开奖、翻卡、稀有度、中奖反馈、空状态与二次购买入口的交互规则。

规范 03

资产管理

统一背包、图鉴、卡牌状态、发货状态和分享场景的操作路径。

规范 04

运营分发

统一 IP 筛选、运营位、活动入口、商家卡片和优惠券展示规则。

06 上线效果与复盘

支付转化明显提升，玩法型消费链路得到验证

线上效果

欧力赏

抽卡机

抽卡机新版于 11 月 18 日上线后，根据产品侧反馈的 11 月 24 日至 11 月 28 日数据，下单后的支付完成效率明显提升。数据不能单独归因于设计，但可以验证“核心信息前置、操作路径简化、优惠感知增强”的优化方向有效。

成功支付 GMV / 下单 GMV

93% 上线前 76%

提升 17 个百分点，说明下单金额中的支付完成比例明显提高。

成功支付人数 / 下单人数

98% 上线前 72%

提升 26 个百分点，说明用户从下单到支付成功的流失被显著压缩。

项目沉淀

2+1 产品 + 规范

完成欧力赏、抽卡机两个产品方向，并沉淀 ACG 交互规范。

这次项目让我更清楚地理解到：ACG 衍生品售卖的核心不是把商品换成随机抽取，而是把 IP 情感、品牌信任、稀缺刺激、购买反馈和社交分享组织成完整体验。只有当用户相信商品价值、理解玩法规则，并能把购买结果继续沉淀和传播时，交易链路才具备持续复购的基础。
